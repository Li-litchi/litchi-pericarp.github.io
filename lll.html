使用<font>标签改变水平线颜色

传统方法与现代CSS方法对比

重要提示

<font>标签在HTML5中已被废弃，不推荐在现代网页开发中使用。本页面仅用于教学目的，展示历史方法。

传统方法：使用<font>标签

在HTML4和更早版本中，可以使用<font>标签设置文本颜色，但这种方法对<hr>元素无效。

尝试使用font标签（无效）

以下代码不会改变水平线颜色：

<font color="red"><hr></font>

如上所示，水平线颜色没有改变。

传统替代方案

在CSS普及之前，开发者使用表格或带颜色的图片创建彩色分隔线：

<table width="100%" cellpadding="0" cellspacing="0">
  <tr>
    <td bgcolor="red" height="2"></td>
  </tr>
</table>

或者使用带颜色的图片：

<img src="red-line.gif" height="2" width="100%">

现代方法：使用CSS

使用CSS可以轻松改变水平线的颜色和样式，以下是几种常用方法：

使用border属性
hr {
  border: none;
  border-top: 3px solid #3498db;
}
使用background-color
hr {
  border: none;
  height: 4px;
  background-color: #e74c3c;
}
使用CSS渐变
hr {
  border: none;
  height: 4px;
  background: linear-gradient(to right, #e74c3c, #3498db);
}
红色 #e74c3c
蓝色 #3498db
绿色 #2ecc71
紫色 #9b59b6
橙色 #e67e22

方法对比

特性 <font>标签方法 CSS方法
HTML标准 HTML4（已废弃） HTML5（推荐）
浏览器兼容性 旧浏览器支持 所有现代浏览器
样式灵活性 非常有限 高度灵活
代码维护 难以维护 易于维护
性能 较差 优秀

CSS方法的优势

  • 符合现代Web标准
  • 样式与内容分离
  • 丰富的样式选项
  • 更好的性能和可维护性
  • 响应式设计支持

<font>标签的缺点

  • HTML5中已废弃
  • 样式与内容混合
  • 有限的样式控制
  • 代码冗余且难以维护
  • 不支持响应式设计

本页面仅用于教学目的，展示HTML历史方法与现代CSS方法的对比。

在实际项目中，请始终使用CSS来设置样式。
